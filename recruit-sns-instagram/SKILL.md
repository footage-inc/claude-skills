## タスク別フロー

### A. 投稿企画の立案（ネタ出し・投稿カレンダー）

1. **2ファイル読み込み**
2. **コンテンツマップ（設計書§4）の5つの柱を確認**:
   - ① 日常の風景 ② 制度・思想の背景 ③ スタッフの声 ④ カルチャー解説 ⑤ 専門性・学び
3. **既存の投稿企画SSがある場合、続きのNo.から作成**
4. **企画設計**:
   - 5つの柱をバランスよく配分（偏らせない）
   - 各企画にペルソナ対応を明記（設計書§4のペルソナ対応列に準拠）
   - 投稿フォーマット（フィード/カルーセル/リール/ストーリーズ）を指定
   - 投稿頻度（設計書§3-2: 火=カルチャー系、木=スタッフ系、土=日常系）を意識
5. **品質チェック**: 設計書§7のチェックリスト全項目を照合

### B. キャプション作成

1. **2ファイル読み込み**
2. **テンプレート適用（設計書§3-3）**:
   ```
   [フック（1行目で止まらせる）]
   [本文：経験ベースのストーリーまたは問いかけ（3〜8行）]
   [ブランドKB参照ポイント（思想を1〜2行）]
   ━━━━━━━━━━━━━━━
   [CTA：自然な誘導]
   ━━━━━━━━━━━━━━━
   [ハッシュタグ：10〜15個]
   ```
3. **文字数**: 150〜300文字
4. **ハッシュタグ**: 設計書§6に準拠（固定5個 + テーマ別5〜10個）
5. **ENJOY with FOOTAGEの使い方**: 設計書§5に準拠（シリーズ名・クロージングの問い）
6. **セルフチェック**: KB§4 Good/Bad例と照合、KB§6アンチパターン確認

### C. 投稿品質チェック（レビュー）

1. **設計書§7の必須チェック全項目を実施**:
   - [ ] ブランドボイスに合っているか
   - [ ] アンチパターンに抵触していないか
   - [ ] ファクトチェック（数字はKB§11と一致か）
   - [ ] 個人情報・利用者情報の映り込みなし
   - [ ] ビジュアルトーン（明るい自然光・笑顔・自然体）
2. **推奨チェックも実施**:
   - [ ] Good/Bad例との照合
   - [ ] ペルソナ対応が明確
   - [ ] 「なぜ」が含まれている
   - [ ] 問いかけが自然

## 絶対に守ること

1. **recruit-brand KBのルールが最上位** — このスキルはKBの下位。矛盾があればKBが優先
2. **投稿品質チェックリスト（設計書§7）は省略不可** — 全投稿で必須チェックを通す
3. **ハッシュタグは10〜15個** — 多すぎるとスパム判定。固定5個+テーマ別5〜10個
4. **キャプション150〜300文字** — 長すぎず、思想を伝えられる範囲
5. **「ENJOY with FOOTAGE」は設計書§5のルールで使う** — 毎回同じフレーズの繰り返しは避ける
6. **曜日テーマを意識** — 火=カルチャー系、木=スタッフ系、土=日常系

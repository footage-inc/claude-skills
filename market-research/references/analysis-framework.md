# 分析フレームワーク

## 基本原則

1. **需要 × 供給 × 競合**の3軸で分析する
2. **現在値 × 将来推計**の時間軸を常に含める
3. **全国平均 × 同県内他圏域**との比較で相対的位置を示す
4. データの限界を明示する（推計の前提条件、カバー外のデータ）

## 分析の切り口

### 1. 需給ギャップ分析

```
需要（将来推計） − 供給（現状のキャパシティ） = ギャップ

ギャップが正 → 供給不足 → FOOTAGE参入/拡大の機会
ギャップが負 → 供給過多 → 差別化 or 別圏域を優先
```

### 2. 時系列トレンド分析

- 2013→2023年の実績推移（病床数・入院患者数・在院日数）
- 2020→2050年の将来推計（5年刻み）
- 変化率で「加速しているか減速しているか」を判断

### 3. 類似圏域ベンチマーク

医療需給総覧p.96に「類似区域との比較」があり、以下の条件で自動マッチングされている:
- 人口規模
- 地域区分（大都市型/地方都市A型 etc.）
- 大学病院の有無

この類似区域との比較で、当該圏域の「異常値」を発見できる。

### 4. 病院タイプ別マッピング

ポジショニングマップの急性期指数による分類:
| 急性期指数 | 病院タイプ | 訪問看護との関係 |
|---|---|---|
| 3.5以上 | 高度急性期病院 | 退院後の在宅移行先として訪問看護を提案 |
| 1.5〜3.4 | ケアミックス病院 | 回復期→在宅の連携パスを構築 |
| 0〜1.4 | 慢性期病院 | 療養病床からの在宅移行先 |

## 分析の限界（必ず明示すること）

1. **訪問看護ステーションの供給データがない**: 医療需給総覧は病院中心。訪問看護ST数・配置看護師数はWAM NET等で別途取得が必要
2. **デイサービスの供給データもない**: 介護事業所検索で補完
3. **発生率は全国一律**: 地域医療構想の推計方法と同様、全国の性年齢別発生率を地域人口に適用。地域固有の疾病構造は未反映
4. **NDBベースの算定回数**: レセプトに計上されないサービスは含まれない
5. **要介護者推計は独自計算**: 介護保険事業計画の推計とは計算方法が異なる
6. **v1.0の初版データ**: 今後のバージョンで指標追加・構成変更の可能性あり

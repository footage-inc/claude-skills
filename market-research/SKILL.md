# 二次医療圏 市場調査スキル

FOOTAGEの経営支援（訪問看護・デイサービス事業）のための市場調査。公的データソースから地域の医療需給構造を分析し、事業判断の根拠を提供する。

## 最初に必ずやること

1. `references/data-sources.md` を読み込む（データソース一覧・取得方法・信頼性評価）
2. `references/analysis-framework.md` を読み込む（分析フレームワーク・指標定義）
3. `references/output-templates.md` を読み込む（出力テンプレート・引用ルール）

## タスク詳細

### タスクA: 医療圏概況分析

**目的**: 対象医療圏の全体像を把握する。

**手順**:
1. 医療需給総覧から対象医療圏のPDFを取得（WebFetch）
2. 以下の指標を抽出・整理:
   - 人口・面積・人口密度・可住地面積比率
   - 2025→2050年の高齢者増減率 × 生産年齢人口増減率（全国330圏域内の立ち位置）
   - 人口10万人あたり: 病院数・診療所数・一般病床数・療養病床数
   - 人口10万人あたり: 医師数・看護師数・PT/OT/ST数
   - 医師偏在指標（圏域内 + 都道府県間比較）
3. 全国平均・同県内他圏域との比較コメントを付与
4. `references/output-templates.md` の「概況レポート」テンプレートで出力

### タスクB: 在宅医療需要予測

**目的**: 訪問看護・在宅医療の中長期需要を定量化する。

**手順**:
1. 医療需給総覧の第3章から以下を抽出:
   - 在宅医療患者数の推計（年齢階級別・居住場所別、2020→2050年）
   - 在宅医療訪問回数（往診料・在宅患者訪問診療料・看取り件数）
   - 要介護者数の推計（年齢別・要介護度別）
2. 補完データの取得:
   - 療養病床の推移（病床数・利用率の減少トレンド）
   - 回復期病床の現状 vs 必要病床数の差分（回復期不足 = 在宅移行加速）
   - 退棟先の構成比変化（「家庭へ退院」「介護施設等へ入所」の推移）
3. FOOTAGE事業へのインプリケーションを導出:
   - 訪問看護利用者の増加率
   - 重度化（要介護3以上）の増加に伴うサービス高度化ニーズ
   - 看取り対応の需要増
4. `references/output-templates.md` の「需要予測レポート」テンプレートで出力

### タスクC: 新規拠点検討

**目的**: 複数の医療圏を比較し、事業展開の優先度を評価する。

**手順**:
1. 比較対象の医療圏（2〜5圏域）を特定
2. 各圏域について以下のスコアリング指標を収集:
   - 需要側: 在宅医療患者数の増加率、要介護者増加率、85歳以上人口増加率
   - 供給側: 療養病床の減少率、回復期病床不足度、医療従事者充足度
   - 競合環境: 訪問看護ステーション密度（※医療需給総覧外のデータ要補完）
   - 地理: 可住地面積比率、人口密度（効率的な訪問が可能か）
3. 比較マトリクス作成 + 総合評価
4. `references/output-templates.md` の「拠点比較レポート」テンプレートで出力

**注意**: 訪問看護ステーション数は医療需給総覧に含まれない。WAM NET or 介護事業所検索で補完する。

### タスクD: 病院連携先調査

**目的**: 退院患者の受け皿として訪問看護を提案できる病院を特定する。

**手順**:
1. 医療需給総覧の第1章・第4章から以下を抽出:
   - 医療機関の指定状況（在宅医療欄=「在後病」指定のある病院）
   - ポジショニングマップ（稼働病床数 × 急性期指数）
   - 入退棟経路の傾向（退棟先の構成比）
   - 救急車受入件数ランキング（患者ボリュームの大きい病院）
   - DPC退院患者の手術シェア × 救急搬送シェア
2. 連携候補の優先度を以下で評価:
   - 急性期指数3.5以上（高度急性期）: 退院後の在宅移行先として最有力
   - ケアミックス病院（急性期指数1.5〜3.4）: 回復期→在宅の連携パスあり
   - 在後病指定あり: 既に在宅医療への意識が高い
   - 退院患者数が多い（DPCシェア上位）: ボリュームが見込める
3. 病院別の連携アプローチ案を付記

### タスクE: 競合環境分析

**目的**: 医療資源の供給充足度を評価する。

**手順**:
1. 医療需給総覧から供給側データを抽出:
   - 人口10万人あたりの各種指標を全国平均と比較
   - 病床機能別の充足度（特に回復期・慢性期の過不足）
   - 診療科別医師数（リハビリテーション科に注目）
   - 類似区域との救急医療提供状況比較
2. 需給ギャップの特定:
   - 回復期病床不足 → 在宅移行ニーズ大
   - 療養病床減少 → 受け皿としての訪問看護需要
   - PT/OT/ST不足 → 訪問リハの差別化機会
3. 供給側の構造変化トレンドをコメント

### タスクF: 採用市場分析

**目的**: 医療従事者の充足度・偏在から採用戦略の前提を整理する。

**手順**:
1. 人口10万人あたり医療従事者数（医師・看護師・薬剤師・PT/OT/ST）
2. 医師偏在指標（圏域内 + 都道府県間）
3. 医療機関別の職員配置（看護職員・PT+OT+STの病院間比較）
4. 全国平均・同県内他圏域との比較で、採用の難易度を推定

### タスクG: 経営支援コンテンツ素材

**目的**: 記事・コラム・提案書に引用可能なファクトデータを抽出する。

**手順**:
1. ユーザーが指定するテーマに関連するデータを医療需給総覧から抽出
2. 引用フォーマットに整形: `出典：医療需給総覧 ○○県△△医療圏（株式会社日本経営）`
3. コンテキストに応じたコメント付き

## データ取得方法

### 医療需給総覧（主データソース）

```
# URL構造
トップ: https://nkgr.co.jp/hsa/
都道府県別: https://nkgr.co.jp/hsa/{prefecture}/
PDF: https://nkgr.co.jp/wp-content/hsa/{region}/hsa{code}-{pref}-{area}.pdf

# 地域コード
region: hokkaido, tohoku, kanto, hokuriku-shinetsu, tokai, kinki, chugoku, shikoku, kyushu
code: 都道府県番号(2桁) + 圏域番号(2桁)  例: 2708 = 大阪府(27) + 大阪市(08)
```

**取得手順**:
1. 都道府県ページをWebFetchしてPDFリンク一覧を取得
2. 対象医療圏のPDFをWebFetchでダウンロード
3. Readツール（pages指定）で必要なセクションを読み取る

**ページ構成の目安**（129ページ中）:
| セクション | 概算ページ | 主な内容 |
|---|---|---|
| 第1章: 地域の概況 | p.3〜21 | 人口・面積・病床地図・医療機関指定状況 |
| 第2章: 医療提供体制 | p.22〜49 | 医師数・看護師数・病床推移・職員数・必要病床数 |
| 第3章: 医療需要の将来推計 | p.50〜77 | DPC推計・疾患別・**在宅医療推計・要介護者推計** |
| 第4章: パフォーマンス | p.78〜128 | ポジショニングマップ・救急・DPCシェア |

**効率的な読み取り**: 全129ページを読む必要はない。タスクに応じて必要なページ範囲のみ読む。

### 補完データソース

医療需給総覧にないデータは以下で補完する:

| データ | ソース | 取得方法 |
|---|---|---|
| 訪問看護ステーション数 | WAM NET 介護事業所検索 | WebSearch |
| 介護事業所一覧 | 介護事業所・生活関連情報検索 | WebFetch |
| 診療報酬・介護報酬 | 厚労省告示 | WebSearch |
| 訪問看護の経営指標 | 経営実態調査（厚労省） | WebSearch |
| 地域医療構想の会議資料 | 各都道府県HP | WebFetch |
| リアルタイム人口統計 | e-Stat | WebFetch |

## 禁止事項

- 医療需給総覧のデータを**出典なしで引用しない**（必ず `出典：医療需給総覧 ○○県△△医療圏（株式会社日本経営）` を付記）
- 将来推計値を**確定値として扱わない**（「推計」「見込み」の文言を必ず付ける）
- 医療需給総覧の供給側データは**病院のみ**。訪問看護ステーション・介護事業所のデータは含まれない。この限界を明示する
- 発生率は**全国一律適用**のため、地域固有の要因（大学病院の存在、交通アクセス等）は未反映。過信しない
- PDFの全部/一部を頒布・提供する行為は利用規約で禁止。データの引用・転載は可

## Gotchas（既知の注意点）

1. **PDF URLの推測は不確実**: 地域コードの命名規則は `hsa{都道府県コード}{連番}-{pref}-{area}.pdf` だが、地域名のローマ字表記にブレがある場合がある。都道府県ページからリンクを取得するのが確実
2. **PDFが6MB超**: WebFetchで取得可能だが大きい。必要ページのみReadすること
3. **v1.0は初版（2026.3公開）**: 今後のバージョンで構成が変わる可能性あり。ページ構成の目安はv1.0基準
4. **在宅医療推計のNDB算定回数ベース**: レセプトベースなので未算定のサービスは含まれない
5. **要介護者推計は介護保険事業計画の推計とは計算方法が異なる**: 都道府県の発生率を用いた独自推計

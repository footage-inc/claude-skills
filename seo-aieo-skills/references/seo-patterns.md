# SEO構成パターン

## 文字数基準

- **目安: 4,000〜5,000文字**（本文のみ。タイトル・CTA・関連リンク除く）
- 3,500文字未満の記事は情報不足。内容を深掘りして加筆する
- 5,500文字を超える場合は冗長な可能性あり。記事分割を検討する
- 経営支援カテゴリは現状500〜600文字の記事が多いため、特に注意して加筆が必要

## SEOチェックリスト

- [ ] 本文が4,000〜5,000文字の範囲内
- [ ] タイトルにメインキーワードが含まれている
- [ ] H2にサブキーワードが含まれている
- [ ] meta_descriptionが120文字以内で記事の要約になっている
- [ ] 導入文の最初の100文字以内にメインキーワードがある
- [ ] 内部リンクが最低1本含まれている
- [ ] CTAセクション（問い合わせ or 関連記事）がある

## 内部リンクルール

- URL形式: `/column/YYYYMMDD_ファイル名/`（訪問看護・経営支援）
- 同カテゴリの関連記事を優先
- 記事末尾「あわせて読みたい」に2-3本
- 本文中にも自然な文脈で1-2本挿入

### 高トラフィック記事から学ぶ構成パターン

GA4 TOP5記事の共通特徴（90日間ユーザー数順、2026-04-08更新）:
1. `/column/2020/`（758users）— **404（削除済み・ファスティング記事）** Googleインデックス残存
2. `/blog/2036/`（632users）— Notion管理対象外
3. `/opening/column/1815/`（472users）— 事業計画書テンプレート ✅ Notion管理済み
4. `/column/1469/`（350users）— **404（削除済み・看護師年収比較）** KB内に原文あり
5. `/column/1655/`（275users）— **404（削除済み・産休育休国際比較）** KB内に原文あり

**重要（2026-04-08 GA4分析結果）:**
- TOP5のうち3件が404。90日で1,383 usersが404ページに着地している
- DB外の404記事は全16件、合計2,375 users/90日の損失
- テーマ一致の旧記事7件は301リダイレクト候補（FSEに再提案要）
- テーマ不一致の9件はGoogle除外待ち（3-6ヶ月で自然除外される見込み）
- `/column/3063/`（31users, HTTP 200）のみNotion DB取込で対応可能

**高PV記事の共通点（生存記事のみ）:**
- タイトルに具体的なベネフィットが含まれている（「テンプレート付」「大解剖」等）
- `/opening/column/1815/` のエンゲージメント率は改善余地あり
- 新規記事のタイトル設計時にこれらを参考にする

### エリアページ × コラム記事の相互リンク設計

GA4でエリアページ（/area/*）が地域SEOで安定集客している:
- /area/kakamigahara/: 161ユーザー, 91.3%エンゲージメント
- /area/chikusa/: 151ユーザー, 89.4%
- /area/moriyama/: 145ユーザー, 84.1%
- /area/anjo/: 140ユーザー, 93.6%

**コラム記事を書く際のリンク設計:**
- 地域に関連するケーステーマの場合、該当エリアページへの内部リンクを追加
- 例: 認知症ケア記事 → 「名古屋市千種区で訪問看護をお探しの方は[千種エリアページ](/area/chikusa/)もご覧ください」
- エリアページ側にも関連コラムへのリンクを追加する（WP手動更新）

### 事業体間クロスリンクルール

現状、訪問看護・経営支援・デイサービスの記事間に相互リンクがほぼない。以下のルールで事業体を横断する回遊を促進する:

| 記事カテゴリ | クロスリンク先 | リンク文脈 |
|-------------|--------------|-----------|
| 訪問看護集客（リハビリ系） | デイサービス集客 | 「通所リハビリとの併用について → Re-moveの取り組み」 |
| デイサービス集客 | 訪問看護集客 | 「在宅でのリハビリ継続 → 訪問看護のリハビリ記事」 |
| 経営支援集客 | 訪問看護集客 | 「利用者目線の記事はこちら → 対応する臨床系記事」 |
| 訪問看護集客 | 経営支援集客 | 「経営者・管理者向けの詳細はこちら → 経営支援コラム」 |

### デバイス別最適化

| デバイス | 構成比 | エンゲージメント率 |
|----------|--------|-------------------|
| Mobile | 66% | 74.5% |
| Desktop | 33% | 80.1% |

モバイルのエンゲージメントがデスクトップより5.6pt低い。記事構成でモバイル体験を意識する:
- 長い段落を避ける（モバイルでは3行以上が読みにくい）
- H2/H3を適切に使い、スクロールでの見出し発見性を高める
- 表はシンプルに（3列以下推奨）、モバイルで横スクロールが発生しない設計

## テクニカルSEO: HTML最適化（2026-04-03 追加）

### Googlebot 2MB制限

GooglebotはHTMLを**最大2MB（非圧縮）**までしか取得しない。超過部分はクロールもレンダリングもされない。外部CSS/JS/画像も各リソースごとに同じ2MB制限が適用される。

> 参照: [Google Search Central Blog (2026-03-31)](https://developers.google.com/search/blog/2026/03/crawler-blog-post)

### footage-nursing.jp への影響

WordPress + SSRの記事ページは通常2MBに達しないため、**現時点で実害リスクは低い**。ただし以下のケースで肥大化しうる:
- テーマ更新でインラインCSS/JSが増加した場合
- ページビルダー系プラグインの導入
- base64エンコード画像の埋め込み

### HTML構成の優先順位ルール

HTMLの先頭に近いほどGooglebotに確実に読まれる。以下の順序を守る:

1. **`<head>`内（最優先）**: title, meta description, canonical, OGP, 構造化データ（JSON-LD）
2. **`<body>`上部**: 本文コンテンツ（H1 → 導入文 → H2...）
3. **`<body>`下部**: ナビゲーション、フッター、装飾要素

### 記事作成時のチェック項目

- [ ] 構造化データ（JSON-LD）は`<head>`内または`<body>`直後に配置
- [ ] インラインCSS/JSを最小限に（外部ファイルに分離すれば別リクエストで取得される）
- [ ] base64画像を本文中に埋め込まない（`<img src>`で外部参照する）
- [ ] 不要なDOMノード（非表示メニュー、装飾用div等）を削減

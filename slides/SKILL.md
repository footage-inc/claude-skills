# 社内勉強会スライド生成スキル

Slidevマークダウン形式で社内勉強会・LT用のスライドを生成する。

## トリガー条件

以下のキーワードで発動:
- 「スライド作って」「スライド生成」
- 「勉強会資料」「LT資料」
- 「プレゼン作成」「プレゼン作って」
- `/slides`

---

## 実行フロー

### Step 1: ヒアリング（最小限）

以下を確認する。ユーザーが既に伝えている情報はスキップ:

1. **テーマ** — 何について話す？
2. **発表時間** — デフォルト: 15分
3. **対象者** — デフォルト: 社内エンジニア
4. **特記事項** — デモあり？コードメイン？概念説明？（任意）

### Step 2: アウトライン提示

スライド生成前に、必ず簡潔なアウトラインを提示して確認を取る:

```
## アウトライン（全12枚・15分想定）

1. タイトル
2. 今日話すこと
3-4. 背景・課題
5-8. 本題（4枚）
9-10. デモ / 実例
11. まとめ
12. Q&A
```

ユーザーが「OK」「いいよ」等で承認したら Step 3 へ。修正指示があれば反映。

### Step 3: スライド生成

`slides.md` を生成する（カレントディレクトリ、またはユーザー指定パス）。

## 出力フォーマット

### フロントマター

```yaml
---
theme: default
title: "{タイトル}"
info: "FOOTAGE 社内勉強会 — {日付}"
author: "{発表者名 or TBD}"
transition: slide-left
fonts:
  sans: 'Noto Sans JP'
  mono: 'Fira Code'
drawings:
  persist: false
---
```

### カスタムスタイル（全スライド共通、1枚目の末尾に挿入）

```html
<style>
:root {
  --slidev-theme-primary: #192864;
  --slidev-theme-accent: #F29A49;
}
h1, h2 { color: #192864; }
a { color: #4A6FA5; }
code { background: #F1F3F5; padding: 0.1em 0.3em; border-radius: 3px; }
.footer { position: fixed; bottom: 1rem; right: 1.5rem; font-size: 0.7rem; color: #868E96; }
</style>
```

### スライド構成テンプレート

| # | レイアウト | 内容 | 目安時間 |
|---|---|---|---|
| 1 | `cover` | タイトル・発表者・日付 | 30秒 |
| 2 | `default` | 今日話すこと（アジェンダ） | 30秒 |
| 3-4 | `default` | 背景・課題・モチベーション | 2分 |
| 5-N | `default`/`two-cols` | 本題（コア内容） | 発表時間の60% |
| N+1 | `default` | まとめ・要点の振り返り | 1分 |
| N+2 | `end` | Q&A / 参考リンク | - |

**スライド枚数の目安:**
- 5分 → 6-8枚
- 10分 → 8-12枚
- 15分 → 12-16枚
- 20分 → 16-20枚
- 30分 → 20-25枚

## デザインガードレール

### 必須ルール

1. **1スライド1メッセージ** — 複数の論点を1枚に詰めない
2. **箇条書きは最大5項目** — 超える場合はスライド分割
3. **コードブロックは15行以内** — 超える場合はハイライト行を指定して要点に絞る
4. **フォントサイズ** — 本文18pt以上を想定（Slidevデフォルトで担保）
5. **色の使用** — FOOTAGEブランドカラー準拠（Navy=#192864, Orange=#F29A49）

### 推奨ルール

- 図解を積極的に使う — Mermaid記法でフロー図・シーケンス図を入れる
- `<v-clicks>` で箇条書きを段階表示にする（聴衆の注意を誘導）
- コードには `{行番号}` ハイライトを付ける
- 各スライドにスピーカーノート `<!-- -->` を付ける（発表メモ）
- セクション切り替え時に `layout: section` スライドを挟む

### やらないこと

- 文字だけのスライドを3枚以上連続させない
- アニメーションの多用（`transition` はグローバル設定の `slide-left` のみ）
- 装飾目的だけの画像挿入
- 見出しなしのスライド

## Mermaid 図の指針

技術トピックでは以下を積極的に使う:

| 図の種類 | 用途 | 記法 |
|---|---|---|
| フロー図 | 処理の流れ、判断分岐 | `graph LR / TD` |
| シーケンス図 | API呼び出し、通信フロー | `sequenceDiagram` |
| クラス図 | データモデル、構造 | `classDiagram` |
| 状態図 | ステータス遷移 | `stateDiagram-v2` |
| ER図 | DB設計 | `erDiagram` |
| ガントチャート | タイムライン | `gantt` |

Mermaidのテーマカラーは以下を指定:

```
%%{init: {'theme': 'base', 'themeVariables': { 'primaryColor': '#192864', 'primaryTextColor': '#fff', 'lineColor': '#868E96', 'secondaryColor': '#F29A49' }}}%%
```

## Slidev記法

詳細は `references/slidev-syntax.md` を参照。主要なもの:

- スライド区切り: `---`
- 2カラム: `layout: two-cols` + `::right::`
- クリック表示: `<v-clicks>...</v-clicks>`
- コードハイライト: ````python {2|3-4|all}```
- スピーカーノート: `<!-- ノート -->`
- Mermaid: ````mermaid ... ```

## 後工程（ユーザーに案内）

スライド生成後、以下を案内する:

```
slides.md を生成しました。

### プレビュー
npx slidev slides.md

### PDF出力
npx slidev export slides.md

### 修正
「3枚目にデモのスクリーンショットを追加して」等、
自然言語で修正指示してください。
```

## 修正対応

ユーザーから修正指示が来たら:
- 該当スライドのみを Edit で修正（全体再生成しない）
- 「もっとシンプルに」→ 文字量を減らし図に置き換え
- 「詳しく」→ スライドを分割して内容を追加
- 「図を足して」→ Mermaid図を追加
- 「順番変えて」→ スライドの並び替え

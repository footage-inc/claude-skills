# Dashboard Design Principles

デジタル庁「ダッシュボードデザインの実践ガイドブック」(2026年3月31日版) に基づくダッシュボード設計原則。
ダッシュボードの設計判断（WHY / WHAT）を担当する。実装手順（HOW）は別スキルが担当。

出典: https://www.digital.go.jp/resources/dashboard-guidebook
PDL1.0 に基づき、内容を要約・加工して利用。

---

## 1. ダッシュボードの類型

| | 提示型 | 探索型 |
|---|---|---|
| 目的 | 概況の把握 | 詳細の分析 |
| 前提知識 | 一般的な知識 | 特定のドメイン知識 |
| 操作 | 単純な操作 | 複雑な操作 |

**原則: 特に指定がなければ提示型で設計する。** 提示型は「見る人が素早く状況を把握して、異常に気づいて、行動の必要性を判断する」ことが目的。フィルターを大量に生やしたり、探索型寄りの設計にしない。

---

## 2. 要件整理: 5W1Hフレームワーク

ダッシュボードを設計する前に以下を定義する:

- **Why** — ダッシュボードの目的は何か？（最上位の目的。変わらない）
- **What, so What** — どんな行動のために、どのような情報を知るべきか？（情報と行動）
- **Who** — 誰がダッシュボードを見るか？（所属、立場、リテラシーのレベル感）
- **When** — いつ見るか？（見るタイミングや頻度）
- **Where** — どこで見るか？（提示する媒体やデバイス）
- **How** — 求められる機能やデータ項目は何か？（必要な更新頻度は？）

---

## 3. 載せるべき情報の原則

ダッシュボードに載せる情報は以下の4つの原則を満たすこと:

1. **目的に則する** — 目的に当てはまるものだけを選ぶ。関係ない情報は取り除く
2. **分解できる** — 全体から部分へと読み解ける単位まで分割する
3. **違いに気づける** — 突出するデータや変化の時期・傾向に気づけるようにする
4. **鮮度が高い** — 最新のデータを反映し、データを定期的に更新する仕組みを構築する

## 4. チャート選択ガイドライン

### 4.1 グラフの種類と用途

| 伝えたい情報 | グラフ | 値の表現 |
|---|---|---|
| 時間変化・傾向 | 折れ線グラフ | 位置 |
| 数量比較 | 棒グラフ | 長さ |
| 時間変化 + 構成比 | 面グラフ | 位置、面積 |
| 構成比 | 円グラフ / ドーナツ | 角度、面積 |
| KPI単値 | 指標（big number） | — |
| 多次元の値の一覧 | 表 | — |

### 4.2 各グラフの選択基準

**折れ線グラフ:**
- 横軸は時間。縦軸の変数が時間以外なら棒グラフを検討
- 系列が複数あるときは系列名を折れ線の近傍に配置
- データ点に丸や四角を配置し、補間の有無がわかるようにする

**棒グラフ:**
- 分類毎の数量を比較するときに使用
- 原点は必ず0にする（誇張や矮小化を防ぐ）
- 系列が複数: 集合棒グラフ or 積み上げ棒グラフ
- 時刻毎の比較には折れ線グラフや面グラフが適する

**面グラフ:**
- 折れ線グラフに塗りを加えたもの。数量と構成比の時間変化を同時に伝える
- 構成比の時間変化のみに着目するなら100%積み上げ面グラフ
- 構成比を伝える必要がないときは折れ線グラフを使用

**円グラフ:**
- 全体の総量が明らかで、構成比をコンパクトに伝える場合に適する
- **多くの場合、棒グラフの方がデータを正確に伝えられる。** 安易に円グラフを選ばない
- 全体の総量が時間変化する場合は面グラフが適する
- ドーナツグラフは中心にくり抜き情報を書き込める

## 5. カラーパレット

| Blue | #D9E6FF | #C5D7FB | #7096F8 | #3460FB | #0017C1 | #000060 |
| Light Blue | #F0F9FF | #C0E4FF | #57B8FF | #008BF2 | #0055AD | #00234B |
| Cyan | #E9F7F9 | #99F2FF | #2BC8E4 | #00A3BF | #006F83 | #003741 |
| Green | #E6F5EC | #9BD4B5 | #51B883 | #259D63 | #115A36 | #032213 |
| Orange | #FFEEE2 | #FFC199 | #FF8D44 | #FB5B01 | #AC3E00 | #541E00 |
| Red | #FDEEEE | #FFBBBB | #FF7171 | #FE3939 | #CE0000 | #620000 |
| Solid Gray | #F2F2F2 | #CCCCCC | #999999 | #767676 | #4D4D4D | #1A1A1A |

### 5.2 カラーパレットの使い方

**テキスト:**
- Text (Black): #000000
- Text (White): #FFFFFF
- Label: #626264
- Link: #0017C1

**背景:**
- Standard: #F8F8FB
- Highlight: #0017C1
- Control: #F1F1F4

**チャート (Primary — Blue系):**
- Blue 1200: #000060 → Blue 900: #0017C1 → Blue 600: #3460FB → Blue 400: #7096F8 → Blue 200: #C5D7FB → Blue 50: #D9E6FF

**セマンティックカラー:**
- Positive: Blue 600 #3460FB / Blue 200 / Blue 50
- Negative: Red 600 #FE3939 / Red 200 / Red 50
- Success: #197A4B
- Error: #CE0000

### 5.3 コントラスト比

- 背景色とグラフ色面のコントラスト比は3:1以上を確保
- 3:1を満たせない場合はグラフの色面領域の近くに数値を記載
- 数値の色は背景色に対して4.5:1以上のコントラスト比を確保

## 6. グラフ設計の原則

### 6.1 知りたいことを知れる

- **シンプルにする** — 冗長なデータや説明、過度に装飾的な表現を取り除く
- **意味のある順列にする** — 数量の大小順、アルファベット順、更新日順など
- **強弱をつける** — 重要な情報には色やサイズ、文字の太さで注意を喚起する
- **待たせすぎない** — 画面表示・操作の反応・反映にかかる時間を短くする

### 6.2 誤解を生まない

- **わかりやすく表記する** — タイトルや系列名、数値の単位を丁寧かつ簡潔に
- **データを定義する** — 何を対象とし、数値の意味は何かをデータ定義として参照可能に
- **表現を歪曲しない** — グラフ軸の値範囲を恣意的に設定しない、値の範囲を省略しない
- **メタ情報を記載する** — データソース、更新日、注釈、免責事項を提供する

## 7. Do's / Don'ts チェックリスト

### Do's（望ましい表現）

1. **全体の指標と詳細のグラフを表示する** — 全体を表す指標を配置し、その詳細のグラフを続ける
2. **グラフ項目の並び順に意味を持たせる** — 数量の大小順、更新日順など比較しやすい順番にする
3. **不要な要素は削除する** — グリッドは最小限、数値と直接関係のないテキスト情報は薄い色に
4. **不要な装飾やリッチな表現は使わない** — 3D表現、ドロップシャドウ等は禁止
5. **グラフに使用する色数を絞る** — 目安は1〜5色。注目すべき系列を明確にする

### Don'ts（望ましくない表現）

1. **タイトルにグラフの内容とデータ種別を表記する** — 「国産自動車」→「国産自動車の出荷台数（月次推移）」
2. **タイトルや凡例をシンプルに保つ** — 端的でない冗長なタイトルは避ける
3. **グラフと凡例を隣接する** — 凡例が離れていたり順番がグラフと逆だったりしない
4. **グラフの原点は原則として0にする** — 特に棒グラフの原点は0。誇張や矮小化を防ぐ
5. **色のみで分類を識別しない** — データ点の形やグラフの塗りの模様など、色以外の情報でも系列を識別可能にする

---

## 8. レイアウトの考え方

- 情報の優先度に応じて目立たせたり、大きさや順番を決める
- 情報量が多い場合はページを分割したり、ナビゲーションを設計する
- 16:9の画面を縦横に2〜6分割できるレイアウトグリッドを使用
- **骨格となるレイアウトを先に検討し、表層の表現を後から詰める**

## 9. チェックリスト

### 利用者の体験
- [ ] 利用者が判断や行動に必要な情報が提供できていますか？
- [ ] データの全体像を捉えるための基準となる指標が掲載されていますか？
- [ ] 本当に必要な情報だけに絞り込めていますか？
- [ ] 利用者に必要以上の操作を求めていませんか？
- [ ] 利用者が迷わず必要な情報まで辿り着けますか？
- [ ] 複数の利用者からのフィードバックを得て改善を重ねましたか？

### 指標と表、グラフのデザイン
- [ ] 利用者が見たいと思っている数値へ優先的に目が向くようになっていますか？
- [ ] グラフや表の配列順序が認知しやすいものになっていますか？
- [ ] グラフと凡例が隣接され、対応づけがわかりやすくなっていますか？
- [ ] 他のデータとの比較や時間的な変化によって、すぐに違いを気づけるようになっていますか？
- [ ] グラフの色数が1〜5色程度に絞れていますか？
- [ ] グラフに含まれる要素（グリッドや枠線）を最小限にしていますか？
- [ ] 不要な装飾や画像、3Dやシャドウのようなリッチな表現は避けていますか？
- [ ] 誰が見てもわかりやすく、誤解のないタイトルや系列名になっていますか？
- [ ] 棒グラフの原点は0になっていますか？
- [ ] データの更新日は記載されていますか？

### データ設計
- [ ] 利用者の目的に見合ったデータだけが選ばれていますか？
- [ ] 利用者が見たいレベルまでデータを分解できるようになっていますか？
- [ ] 利用者が必要な頻度でデータが更新されるような仕組みや運用プロセスになっていますか？
- [ ] 数量単体だけでは判断が困難な場合、理解を促すために比較できる数値が掲載されていますか？
- [ ] データ定義を参照できるようになっていますか？

### アクセシビリティ
- [ ] 色だけでグラフの情報を識別していませんか？
- [ ] グラフに代替テキスト（alt情報）を付与していますか？
- [ ] データファイル（ExcelやCSV）を公開していますか？（公開ダッシュボードの場合）

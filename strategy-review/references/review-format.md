# 出力フォーマット・品質原則

## 各エージェントの出力フォーマット

SEOスキルの「テーマ = トピック × 独自アングル × Footage実体験」を施策検討に転用。

```markdown
## [エージェント名] の見解

### 問い（このエージェントが立てる問い）
- この施策について、[視座]の観点から問うべきことは何か？

### 分析（学術理論 × Footageの文脈）
- **理論的根拠**: [引用する理論・思想家と、なぜこの理論が関連するか]
- **Footage実体験との接続**: [KBから引ける具体例、または類似の過去経験]
- **現状とのギャップ**: [理想と現実の差分]

### 判定
- **推進** / **条件付き推進** / **保留** / **再考**
- 判定理由（1〜2文）

### 提案（あれば）
- 施策をより良くするための具体的な修正案
```

## 品質原則（SEOスキルから継承）

### 1. エビデンスベース
- 「〜と言われている」禁止
- 理論名・思想家名を明示
- Footage実体験を引く場合も「KBの○○から」と出典を示す

### 2. 独自アングル必須
- 一般的な経営コンサルの回答で終わらない
- Footage固有の文脈（自律分散型、臨床推論文化、DX活用）に必ず接続
- 「どの訪問看護ステーションでも同じことが言える」レベルの分析は不可

### 3. 3軸構造
- **トピック**: 施策そのもの
- **独自アングル**: 各エージェントの理論的視座
- **Footage実体験**: KBから引ける具体例

### 4. 逆説・反常識の重視
- 批評家だけでなく、全エージェントが「この施策の裏側」を意識
- 全員が推進の場合、最後に「全員一致の時こそ疑うべき点」を検討

### 5. フィードバックループ
- 施策実行後、結果を振り返って知見を蓄積
- `improvement-log/` に施策の事後レビューを保存（将来拡張）

# ia-framework — 情報設計フレームワーク

設計意図（「状態変化を見せたい」「不安を和らげたい」）を、具体的なUIパターンに翻訳する。
デザイナーの暗黙知——「このデータならこのUI」——を3軸の形式知に変換する。

**参照必須:** `~/dev/footage-aix/DESIGN.md` のデザイントークンを常に参照すること。

---

## 設計原則: 相対化と行動惹起

全てのUI設計判断の上位にある2つの原則。3軸診断やパターン選択より先に、これを満たしているかを問う。

### 相対化 — 数字を「意味」に変える

裸の数値は情報ではない。比較対象があって初めて意味になる。

| 相対化の軸 | 例 | 実装パターン |
|-----------|-----|------------|
| **時間比較** | 先月比、前年同期比、推移 | 折れ線グラフ、差分Badge(▲+12%)、スパークライン |
| **目標比較** | 目標CPA vs 実績CPA、充足率 | プログレスバー、閾値ライン、達成率% |
| **他者比較** | 拠点間、職種間、業界平均 | グループ棒グラフ、ランキング、ヒートマップ |
| **基準比較** | 正常/異常、36協定閾値 | 色変化(緑→赤)、アラートBadge、Pulse |

**チェック:** 全てのT1(単値KPI)セクションに対して「何と比べているか？」を問う。比較対象がなければ、そのKPIは意味を持たない。追加するか、表示しない。

### 行動惹起 — 見たら動ける設計

情報を見せて終わりにしない。「で、何をすべきか」まで設計する。

| 行動惹起のレベル | 説明 | 実装パターン |
|----------------|------|------------|
| **L0 気づき** | 異常に気づける | Pulse、色変化、カウンター |
| **L1 理解** | 何が問題か分かる | ドリルダウンリンク、ツールチップ、要因分解 |
| **L2 判断** | 何をすべきか決められる | 推奨アクション表示、優先順位ソート |
| **L3 実行** | その場で行動できる | インラインアクションボタン、ステータス変更、ワンクリック操作 |

**チェック:** 全てのセクションに対して「見た後にユーザーは何をする？」を問う。行動が想定できないセクションは設計を見直す。L0止まりのセクションが多い画面は「見て終わり」になる。最低1つはL2-L3まで到達させる。

### 説明不要 — シンプルで美しいUX

UIが「使い方」を説明し始めた時点で負けている。見た瞬間に分かる設計を目指す。

| 原則 | やること | やらないこと |
|------|---------|------------|
| **視覚的自明性** | アイコン+色で状態が伝わる | ツールチップに頼って意味を説明する |
| **認知負荷の最小化** | 1セクション1メッセージ | 複数のメトリクスを1カードに詰め込む |
| **ノイズの排除** | 今の判断に必要な情報だけ表示 | 「念のため」で項目を追加する |
| **余白の意図** | グルーピングと階層を余白で表現 | 線や枠で区切りを明示する |
| **一貫性** | 同じ意味には同じパターン・色 | 同じステータスを画面ごとに別デザインにする |

**チェック:**
- 初見のユーザーがヘルプなしで使えるか？
- 各セクションを5秒で理解できるか？ → できないなら情報を減らすか分割する
- 装飾的な要素を全て削除しても機能するか？ → するなら装飾は不要

### Blueprint生成時の必須プロセス

```
STEP 0: 設計原則チェック（全セクションに対して）
  ├── 相対化: このKPIは何と比較しているか？ → 比較対象なしなら追加
  ├── 行動惹起: この情報を見た後ユーザーは何をする？ → 行動なしなら再設計
  ├── 行動惹起: L2-L3行動に到達するセクションが最低1つあるか？ → なければ追加
  ├── シンプル: 初見で5秒以内に理解できるか？ → できないなら情報を減らす
  └── シンプル: 装飾を全て消しても機能するか？ → するなら装飾は不要
STEP 1: 情報理解 — 各セクションを3軸で診断
STEP 2: UIパターンマッピング — 3軸の掛け合わせでパターン決定
STEP 3: Blueprint生成 — 構造化された設計仕様書を出力
```

## STEP 1: 情報理解 — 3軸診断

各画面セクションに対して以下の3軸を特定する。

### 軸1: 情報タイプ (9種)

| ID | 情報タイプ | 定義 | FOOTAGE例 |
|----|-----------|------|-----------|
| `T1` | **単値KPI** | 1つの数値で状態を示す | 募集中オーダー数、本日の記録数 |
| `T2` | **ステータス** | 離散的な状態を示す | 候補者ステータス(書類選考/面接/内定)、オーダー状態(open/filled) |
| `T3` | **時系列** | 時間に沿った値の変化 | 月別応募数推移、売上推移 |
| `T4` | **比較** | 複数カテゴリの値を並べる | 拠点別オーダー数、職種別CPA |
| `T5` | **構成比** | 全体に対する各部分の割合 | ステータス別候補者比率、雇用形態別構成 |
| `T6` | **一覧/コレクション** | 同型のアイテムの集合 | 候補者一覧、オーダー一覧、トピックス |
| `T7` | **進捗** | 完了/目標に対する現在地 | 同期率、充足率、目標達成率 |
| `T8` | **異常/アラート** | 閾値を超えた逸脱 | 期限超過、停滞候補者、36協定超過 |
| `T9` | **ナラティブ** | 文脈説明・ガイダンス | ウェルカムメッセージ、空状態ガイド、ヘルプ |

### 軸2: 認知タスク (5種)

| ID | 認知タスク | ユーザーの頭の中 | 典型的な問い |
|----|-----------|----------------|------------|
| `C1` | **把握する** | 全体像を素早く掴みたい | 「今どうなってる？」 |
| `C2` | **監視する** | 異常や変化を見逃したくない | 「ヤバいことが起きてないか？」 |
| `C3` | **比較する** | 複数の値を並べて判断したい | 「どっちがいい？どこが違う？」 |
| `C4` | **探索する** | 条件を変えて深掘りしたい | 「なぜこうなった？もっと詳しく」 |
| `C5` | **行動する** | 次のアクションを決めて実行したい | 「何をすべき？どれを選ぶ？」 |

### 軸3: コンテキスト (4種)

| ID | コンテキスト | 説明 | 設計への影響 |
|----|-------------|------|------------|
| `X1` | **朝の確認** | 1日の始まりに全体を確認 | 提示型、情報密度低め、ポジティブトーン |
| `X2` | **業務中の作業** | タスク実行中に参照 | 探索型OK、情報密度高め、効率重視 |
| `X3` | **異常発生時** | 問題を検知して対処 | アラート目立たせる、アクション明示 |
| `X4` | **報告・共有** | 他者に状況を伝える | 印刷/エクスポート考慮、解釈を添える |

## STEP 2: UIパターンマッピング

3軸の掛け合わせで、UIパターンが**ほぼ一意に決まる**。

### マッピングテーブル（情報タイプ × 認知タスク → UIパターン）

| | C1 把握 | C2 監視 | C3 比較 | C4 探索 | C5 行動 |
|---|---------|---------|---------|---------|---------|
| **T1 単値KPI** | StatCard | StatCard+Pulse | StatCard横並び | StatCard→ドリルダウン | StatCard+CTAリンク |
| **T2 ステータス** | Badge | Badge+Pulse | Badge一覧(横並び) | フィルタ付きBadge | Badge+アクションボタン |
| **T3 時系列** | 折れ線グラフ | 折れ線+閾値ライン | 複数系列折れ線 | 期間フィルタ付き折れ線 | 折れ線+アノテーション |
| **T4 比較** | 棒グラフ(横) | 棒グラフ+閾値ライン | グループ棒グラフ | ソート可能テーブル | 棒グラフ+ハイライト行 |
| **T5 構成比** | ドーナツ+中央値 | 100%積み上げ棒 | 複数ドーナツ並列 | 積み上げ面グラフ | 構成比+アクション |
| **T6 一覧** | コンパクトリスト | リスト+Pulse行 | テーブル(ソート可) | フィルタ+検索+テーブル | リスト+行アクション |
| **T7 進捗** | プログレスバー | プログレス+色変化 | 複数プログレス並列 | 詳細内訳テーブル | プログレス+残タスク |
| **T8 異常** | アラートバナー | Pulse+カウンター | 異常一覧テーブル | フィルタ付き異常一覧 | アラート+即時アクション |
| **T9 ナラティブ** | テキストカード | — | — | 折りたたみ詳細 | テキスト+CTAボタン |

### コンテキスト補正ルール

| コンテキスト | 補正 |
|-------------|------|
| X1 朝の確認 | StatCardを大きめに。リストは最新5件に絞る。色はソフトに |
| X2 業務中 | 情報密度上げてOK。フィルタ・ソート・検索を有効化 |
| X3 異常発生 | Pulse必須。アラートカラー(Coral/Red)を使用。アクションボタン目立たせる |
| X4 報告 | グラフにタイトル・単位・出典を付ける。解釈テキストを添える |

### パターン1: ダッシュボード Welcome + Stats
- **3軸:** T9ナラティブ×C1把握×X1朝の確認 → テキストカード(グラデーション背景)
- **3軸:** T1単値KPI×C2監視×X1朝の確認 → StatCard+Pulse(6カード2行グリッド)

- **実装:** `agent-portal/src/app/dashboard/orders/OrderMatrixView.tsx`

### パターン3: 候補者パイプライン
- **3軸:** T6一覧×C4探索×X2業務中 → フィルタ+検索+テーブル
- **3軸:** T2ステータス×C5行動×X2業務中 → Badge+アクションボタン(ステータス変更モーダル)
- **実装:** `agent-portal/src/app/dashboard/candidates/page.tsx`

### パターン4: Kaipoke同期ダッシュボード
- **3軸:** T1単値KPI×C1把握×X2業務中 → StatCard(大:4xl)
- **3軸:** T2ステータス×C1把握×X2業務中 → Badge(3カラム色分け: draft/queued/synced)
- **3軸:** T7進捗×C2監視×X2業務中 → プログレスバー(アニメーション付き)
- **実装:** `projects/kaipoke-app/app/src/pages/DashboardPage.tsx`

### パターン5: 空状態
- **3軸:** T9ナラティブ×C5行動×ANY → テキスト+CTAボタン(EmptyState)
- **判断理由:** データがないとき、ユーザーを次のアクションに導く
- **実装:** `agent-portal/src/components/ui/EmptyState.tsx`

## Gotchas

- **円グラフを安易に選ばない。** 棒グラフの方が正確に伝わる場面が多い（デジタル庁ガイドライン）
- **提示型と探索型を混ぜない。** 1画面に両方入れると認知負荷が跳ね上がる。モーダルで分離する
- **Pulseの乱用禁止。** 本当に注意が必要な箇所(3つ以下)にのみ使う。全部光ると何も光らない
- **ステータスは色+アイコンの二重符号化。** 色覚多様性対応（WCAG）
- **日本語テキストのline-height 1.8を守る。** 1.5以下は可読性が著しく低下する
- **StatCardは6枚以下。** 7枚以上は認知限界を超える。グルーピングして2段に収める
